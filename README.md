# Tetris
테트리스

## 게임 흐름
1. 시작 시 플레이
2. 점수에 따라 레벨 및 속도 변경
3. 블록 생성 실패 시 게임 종료

## 구현

### UI
IDrawable: 그리는 객체라면 해당 클래스 상속, Draw::CreateDrawable로만 생성하여 CRenderer의 그리는 대상 배열에 저장

CPen, CBrush: 그리기 용 Pen 및 Brush RAII

CRenderer: HDC, Hwnd 등 제공 용 및 전체적인 그리기 관리

### 게임 로직
CTetrisManager: 전체적인 테트리스 관련 로직 처리

CBoard: 테트리스가 일어나는 보드 UI. 프리뷰 박스, 점수, 게임 오버 등 표기

CTetromino: 블록 UI, 타입 별 회전에 대한 전체 블록 값을 [8][4][4] 배열로 보관

틱은 QueryPerformance 및 QueryCounter를 통해 구현.

### 점수 계산
1000점 당 1레벨, 레벨 당 드랍 시간 2 → 0.1f까지 조정

## 네트워크
현재 로컬 서버 클라이언트만 구현 상태입니다.

추후 다른 네트워크에 있는 클라이언트와 연결하는 기능도 추가하도록 하겠습니다.

### CNetworkManager
호스트와 연결자로 적용

첫 연결 시 Seed값을 주고 받아 동일한 블럭 순서로 적용

각자 현재 틱 순서를 보내서 틱 동기화 하도록 반영 (락스텝)

### CPlayerSession
기존 하나만 있던 테트리스 판을 두 개로 그리기 위해 추가한 클래스

## 리플레이

### CReplayManager
리플레이 관련 기능을 적용하는 클래스 다음 내용을 저장합니다

시드 값, 플레이 한 유저 수, 틱 별 유저의 동작

실행 폴더의 Replay 폴더 내에 저장되며, 리플레이 버튼 클릭 시 현재 해당 폴더에 있는 .rep 파일을 가져와 목록으로 보여줍니다.
